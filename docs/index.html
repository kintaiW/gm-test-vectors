gm-testgen

国密算法（SM2 / SM3 / SM4）正确性测试向量生成与验证 CLI 工具

本项目仅供学习和开发测试使用，请勿用于商用密码产品送检现场测试

核心能力

生成测试向量

调用 gm-sdk-rs 加密/签名 API，随机生成测试数据，经过「生成 → 自验证」循环后写入文件，确保每条向量的正确性。

验证测试向量

读取指定目录下的向量文件，根据文件名自动识别算法和模式，逐条调用密码算法进行反向验证。

多平台发布

GitHub Actions 自动编译 Linux / Windows / macOS（x64 + ARM）四个平台，推送 tag 即发布到 Releases。

CLI 用法

# 生成全部算法的测试向量（默认 10 组）
$ gm-testgen generate

# 指定算法、数量和输出目录
$ gm-testgen generate --algo sm4 --count 20 --output ./my-vectors

# 验证指定目录下的测试向量
$ gm-testgen verify --input ./output

generate 参数：--algo（sm2 | sm3 | sm4 | all）、--count（数量）、--output（输出目录）
verify 参数：--input（向量文件目录）

文件格式

纯文本 key= value 格式，记录间以空行分隔。数值为十六进制编码。
